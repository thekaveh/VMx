# ADR 0046 — v2.6 spec-accuracy clarification (ch. 14 `IPageable.PageCount`)

**Status:** Accepted (2026-06-22)
**Spec version:** 2.6.0

## 1. Context

A maintenance sweep found a self-contradiction in chapter 14 §2.10 (the
`IPageable` capability sketch). The `PageCount` comment read
`# derived: ceil(itemCount / PageSize)`, which divides by zero when
`PageSize == 0` and contradicts both:

1. the adjacent `PageSize` comment one line above — "0 means 'all items in one
   page'" (i.e. `PageCount == 1` when `PageSize == 0`); and
1. the canonical implementation spec in chapter 21 §5.1
   (`PagedComposition<TVM>`), which states
   `ceil(source.Count / PageSize), or 1 when PageSize == 0`.

All three full-parity implementations return `1` when `PageSize == 0` (e.g.
`langs/typescript/src/collections/pagedComposition.ts` — `if (this.#pageSize === 0) return 1;`; the C# and Python `PagedComposition` do the same). The ch.14
sketch was the only place the `PageSize == 0` case was dropped from the
formula.

## 2. Decision

Chapter 14 §2.10's `PageCount` comment is corrected to
`# derived: ceil(itemCount / PageSize), or 1 when PageSize == 0`, matching
chapter 21 §5.1 and the implementations.

## 3. Rationale

- Same principle as ADR-0044/0045: where the prose is self-contradictory or
  incomplete relative to the canonical chapter and the implementations, the
  prose is the defect. Correcting the comment preserves the intended behavior
  and cross-flavor parity.
- No code changes, no conformance-ID changes, no version bump: the
  clarification describes existing 2.6.0 behavior accurately.

## 4. Consequences

- A reader of chapter 14 §2.10 no longer derives a divide-by-zero for the
  `PageSize == 0` ("all items in one page") case; the capability sketch is now
  consistent with chapter 21 §5.1 and with `IsPagingEnabled = PageSize > 0`.
