Notes-Showcase VM Hierarchy

scenario contract from spec/proposals/2026-05-29-notes-showcase-scenario.md — applies identically across C# Avalonia, Python Textual, TypeScript React, and SwiftUI (names shown in their canonical scenario form per ADR-0006)

Root composition

  • • WorkspaceVM is the only AggregateVM6 in the example — its arity (6) is the reason VMx absorbed ADR-0034 in v2.2
  • • The 6 slots are heterogeneous: a tree, a paged list, a form, a status text, an overlay group, and a capability bar — no two share a common VM type
  • • WorkspaceVM is also where every cross-pane Rx subscription is wired (notebooks→notes, notes→form, notes→capabilities)

3-pane data flow

  • • Left → center: NotebooksRoot.Current change → load notes via repo → set NotesView.Items
  • • Center → right: NotesView.Current change → NoteForm.BindTo(current) (R3-Crit2; handles null per R4-Imp1)
  • • Center → top: NotesView.Current's capabilities → CapabilityActionsVM action bar enabling
  • • Bottom (StatusBar) is a derived property — pure function of the other VMs' state

Notification flow

  • • NoteVM.DeleteCommand wraps in ConfirmationDecoratorCommand (R3-Crit1 parity)
  • • Notebook + Note CRUD commands post to INotificationHub
  • • NotificationsVM subscribes to the hub and renders toast (NotificationVM) + confirm (ConfirmationVM) overlays
  • • Opt-in: a worker that disables the sub-package gets a null-impl that auto-approves confirmations

Capability + CRUD layer

  • • InMemoryRepository implements 4 CRUD micro-interfaces (ICreatable, IReadable, IUpdatable, IDeletable)
  • • Same repo surface is consumed by all 3 panes — single source of truth
  • • Each example-app VM declares only the capabilities it actually offers; CapabilityActionsVM reads them off NotesView.Current and enables/disables actions accordingly

Threading + lifecycle

  • • All cross-pane subscriptions go through Dispatcher.foreground via observe_on (R4-Imp2)
  • • Filter before marshal — preserved across all 4 flavors (R5-Imp)
  • • NoteForm resets TagDraft on unbind so the next note's editor starts clean (R5-Min)
  • • BindTo subscription disposes the old binding when NotesView.Current moves (R3-Imp)

Pure-VM contract

  • • View layers across all 4 flavors are declarative-only — no useState, no _compute_*, no axaml code-behind beyond InitializeComponent()
  • • CI gates: check-axaml-codebehind.py (C#) · check-textual-views.py (Py) · check-layer-imports.py (all)
  • • Parity gate: check-showcase-parity.py — every WorkspaceVM child has a test file in all 4 flavors
