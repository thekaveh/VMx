VMx — Library Class Map

Cluster-level view of every class family · spec/VERSION 2.6.0 · five bands

Band 1 · Lifecycle base

  • • Every VM derives from ComponentVMBase / _ComponentVMBase
  • • ConstructionStatus enum: 6 states · 13 legal transitions
  • • ConstructionStatusChangedMessage + PropertyChangedMessage are bus envelopes
  • • Fixture-backed transitions: spec/fixtures/lifecycle-transitions.json

Band 2 · VM family

  • • Leaf: ComponentVM · ComponentVM<M> · ReadonlyComponentVM<M>
  • • Composite: CompositeVM<VM> · CompositeVM<M, VM> (v2.6 builder hooks ADR-0042) · GroupVM<VM> · HierarchicalVM<TM, TVM> · PagedComposition<VM>
  • • Aggregate: AggregateVM1..6 (arity 6 from ADR-0034)
  • • Decorators: ForwardingComponentVM · ForwardingCompositeVM
  • • Specialized: FormVM · NotificationVM · ConfirmationVM (band 5)

Band 3 · Commands & capabilities

  • • Commands: RelayCommand · CompositeCommand · DecoratorCommand chain · ConfirmationDecoratorCommand · ModeledCrudCommands<M,VM>
  • • 22 capability micro-interfaces · CAP-001..022 · spec/14 · ADR-0010
  • • Selection · Expansion · Lifecycle · Query · Dialog · CRUD · Form · Hierarchy
  • • Composed via dependency, not inheritance

Band 4 · Services + data flow

  • • Services: MessageHub · RxDispatcher · ILocalizer · IDialogService (v2.1) — every service has a Null* sibling per ADR-0017
  • • State helpers: DerivedProperty · SearchableState · ExpandableState
  • • Collections (v2.1): ServicedObservableCollection · ObservableList · ObservableDictionary (multi-key) · PagedComposition
  • • Builders: fluent, immutable, validate on Build() · v2.6 modeled/factory hooks (ADR-0042)
  • • Tree: walk · find · walk_expanded

Band 5 · Notifications + specialized VMs

  • • Opt-in sub-package: vmx.notifications / VMx.Notifications / @thekaveh/vmx/notifications
  • • INotificationHub + NullNotificationHub + ConfirmHelper
  • • NotificationVM · ConfirmationVM (render-side VMs)
  • • FormVM (snapshot/revert lifecycle, v2.1+)
  • • Bridges ConfirmationDecoratorCommand (Band 3 ↔ Band 5)

Per-flavor surface

  • • C# uses PascalCase (e.g. ComponentVM<M>); Python uses snake_case (ComponentVMOf[M]); TS & Swift use camelCase
  • • ComponentVMOf exists because Python/TS/Swift generics syntax can't overload the unparameterized name (ADR-0006)
  • • Public method names follow the per-language idiom (IsValid / is_valid / isValid)
  • • Collection "Count" message channel is a spec-literal exception (per ADR-0024)
