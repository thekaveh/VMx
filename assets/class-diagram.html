VMx Class Diagram

spec v2.4.0 — class families, relationships, and the contracts each band implements (language-neutral; PascalCase / snake_case / camelCase per ADR-0006)

1. Lifecycle base

  • • Every VM (leaf, composite, group, aggregate) derives from ComponentVMBase
  • • Holds the injected MessageHub + Dispatcher (ADR-0003)
  • • Public dispose() orchestrates: status flip → _on_dispose() hook → Subject cleanup
  • • ConstructionStatus drives the 5-state machine that LIFE-001..013 specifies

2. VM family (5 idioms)

  • • Leaves — single VM; modeled variants hold Readonly<M>
  • • Composites — homogeneous children, one Current slot
  • • Groups — homogeneous peers, no selection, batch-aware
  • • Aggregates — heterogeneous tuple slots, arity 1..6 (v2.2 adds 6 per ADR-0034)
  • • Specialized — FormVM, notification rendering VMs, forwarding decorators

3. Commands & capabilities

  • • RelayCommand / RelayCommand<T> — predicate + task + reactive triggers
  • • DecoratorCommand chain: Confirmation, Logging, user-defined
  • • ModeledCrudCommands<M, VM> — Create / Read / Update / Delete bundle
  • • 22 capability micro-interfaces — Selection / Expansion / Lifecycle / Query / Dialog / CRUD
  • • Concrete VMs implement only what they offer; the host UI binds against the interfaces it sees

4. Services · Messages · State · Collections

  • • Services are constructor-injected; every one has a NullX sibling (ADR-0017)
  • • Messages flow through the hub; subscribers consume via Rx
  • • State helpers (Searchable, Expandable, Derived) package recurring patterns
  • • Builders are fluent immutable; Build() validates the spec contract
  • • Collections (v2.1) — hub-aware, granular-eventing, multi-key, batchable

5. Notifications sub-package

  • • Ships separately (C# assembly · Python sub-package · TS sub-path)
  • • INotificationHub hosts async Post / Resolve / Pending flow
  • • ConfirmHelper bridges to ConfirmationDecoratorCommand in Band 3
  • • Rendering VMs (NotificationVM, ConfirmationVM) live in Band 2 but are bound here
  • • Opt-in: no core dependency on this package; null-impl always safe
