# Examples

VMx ships a small set of examples with two jobs: prove the minimal surface in
small demos and prove the full cross-language contract in the flagship Notes
Workspace portfolio.

## Start Here

- [Notes Workspace](notes-workspace.md) for the full stable four-flavor scenario.
- [VM Layer Map](notes-workspace-vm-layer.md) when you want the component-to-VM
  mapping first.
- [Smaller Examples](smaller-examples.md) for console, WPF, tkinter, and
  inspector demos.

## Current Portfolio

| Flavor     | Small demos                                          | Flagship app             |
| ---------- | ---------------------------------------------------- | ------------------------ |
| C#         | Console `HelloVMx`, WPF Todo                         | Avalonia Notes Workspace |
| Python     | Console `hello_vmx`, tkinter Todo, Textual Inspector | Textual Notes Workspace  |
| TypeScript | Console `hello-vmx`                                  | React Notes Workspace    |
| Swift      | none beyond the flagship today                       | SwiftUI Notes Workspace  |
| Rust       | Console `hello-vmx`                                  | no UI flagship yet       |

## Reading Strategy

- Use the smaller examples when you want the minimum lifecycle, hub, and
  builder shape.
- Use Notes Workspace when you want pagination, search, dialogs,
  notifications, theme state, or cross-flavor parity.
- Use the source READMEs for full run commands and per-project layout:
  [C# examples](https://github.com/thekaveh/VMx/blob/main/examples/csharp/README.md),
  [Python examples](https://github.com/thekaveh/VMx/blob/main/examples/python/README.md),
  [TypeScript examples](https://github.com/thekaveh/VMx/blob/main/examples/typescript/README.md),
  [Swift flagship](https://github.com/thekaveh/VMx/blob/main/examples/swift/notes-showcase/README.md),
  [Rust examples](https://github.com/thekaveh/VMx/blob/main/examples/rust/README.md).
